# empresa-shopeasy

## Setor
E-commerce

## Contexto do problema
A ShopEasy possui entregas frequentes para front-end e back-end, porém o fluxo de CI/CD não está documentado de forma centralizada. Isso gera incerteza sobre critérios de promoção entre ambientes, responsabilidades de aprovação e estratégia de rollback.

## Objetivo do desafio
Produzir documentação de pipeline CI/CD com visão ponta a ponta, incluindo gatilhos, estágios, controles de qualidade e operação em caso de falhas.

## Instruções resumidas para os grupos
1. Faça um fork deste repositório para a conta do grupo.
2. Trabalhe exclusivamente no fork do grupo.
3. Preencha a documentação na pasta `docs/`, usando `docs/pipeline-template.md` como referência e consolidando o resultado em `docs/pipeline.md`.
4. Entregue o link do fork do grupo conforme orientação da disciplina.
